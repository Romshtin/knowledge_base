# Knowledge Base

Локальная цифровая база знаний в формате Markdown с семантическим поиском через MCP-сервер для Claude Code.

## Структура

```
knowledge_base/
├── .mcp/                  # MCP-сервер для семантического поиска
│   ├── server.py          # сервер (SentenceTransformers + LanceDB)
│   ├── requirements.txt   # зависимости Python
│   ├── run_server.cmd     # запуск сервера для Claude Code
│   ├── run_mcp_server.cmd # альтернативный запуск
│   └── README.md          # инструкция по настройке MCP
├── .kb_index/             # векторный индекс (генерируется автоматически)
├── books/                 # исходники книг (локально, не в git)
└── README.md              # этот файл
```

> **Важно:** папка `books/` содержит защищённые авторским правом материалы и исключена из репозитория через `.gitignore`. Она используется только для локального индексирования и личного чтения.

## Быстрый старт

1. **Клонируй репозиторий** и перейди в папку:
   ```powershell
   cd knowledge_base
   ```

2. **Установи зависимости MCP-сервера:**
   ```powershell
   cd .mcp
   py -m pip install -r requirements.txt
   cd ..
   ```

3. **Скачай модель эмбеддингов** `all-MiniLM-L6-v2` в кеш HuggingFace (`%USERPROFILE%\.cache\huggingface\hub`). Сервер работает в офлайн-режиме и ожидает модель локально.

4. **Подключи MCP-сервер к Claude Code** (см. `.mcp/README.md`).

5. **Добавляй свои заметки (`.md`)** — они автоматически проиндексируются при следующем запуске сервера.

## Технологии

- **LanceDB** — векторная база данных без сервера.
- **SentenceTransformers** (`all-MiniLM-L6-v2`) — эмбеддинги для семантического поиска.
- **MCP (Model Context Protocol)** — интеграция с Claude Code.

## Известные нюансы

- Сервер использует офлайн-режим (`HF_HUB_OFFLINE=1`, `TRANSFORMERS_OFFLINE=1`), чтобы избежать попыток скачивания модели и падений на Windows.
- Совместимые версии библиотек зафиксированы в `.mcp/requirements.txt`.
- При изменении Markdown-файлов сервер автоматически переиндексирует их по MD5-хешу.

## Безопасность

- Никакие секреты, IP-адреса, токены или персональные данные не хранятся в репозитории.
- Защищённые авторским правом материалы исключены из Git через `.gitignore`.
